The Vinyl Concierge Firestore Schema

Firebase Auth identifies signed-in customers and the owner. Firestore stores private customer sessions, public catalog records, de-identified demand signals, owner-only operations, and a public read-only configuration document for the recommendation kill switch. 

erDiagram
  AUTH_USER ||--|| USERS : "creates profile"
  USERS ||--o{ USER_SESSIONS : "owns private sessions"
  USER_SESSIONS ||--o{ DEMAND_SIGNALS : "emits de-identified demand"
  ALBUMS ||--o{ DEMAND_SIGNALS : "referenced by albumId"
  USERS ||--o{ INVENTORY_EVENTS : "owner writes"
  USERS ||--o{ OWNER_ACTIONS : "owner writes"
  USERS ||--o{ GOVERNANCE_EVENTS : "owner writes"
  CONFIG ||--o{ USER_SESSIONS : "recommendationsEnabled gates"

  AUTH_USER {
    string uid PK
    string email
    string displayName
    string photoURL
  }

  USERS {
    string uid PK
    string email
    string displayName
    string photoURL
    string role "customer or owner"
    timestamp createdAt
    timestamp lastActiveAt
  }

  USER_SESSIONS {
    string sessionId PK
    map input "artists, genres, mood, context"
    array recommendations
    number collectionCoverageScore
    timestamp createdAt
  }

  DEMAND_SIGNALS {
    string signalId PK
    string albumId FK
    string artist
    string genre
    string signalType "rec_request"
    number weight
    timestamp createdAt
  }

  ALBUMS {
    string albumId PK
    number discogsReleaseId
    string title
    string artist
    number year
    array genres
    array styles
    array moodTags
    array contextTags
    string thumbUrl
    boolean inStock
    number stockCount
    number price
    string source
    timestamp discogsFetchedAt
  }

  INVENTORY_EVENTS {
    string id PK
    string albumId FK
    string action
    number quantity
    string note
    timestamp createdAt
  }

  OWNER_ACTIONS {
    string id PK
    string actionType
    string status
    string rationale
    timestamp createdAt
  }

  GOVERNANCE_EVENTS {
    string id PK
    string eventType
    string severity
    string ownerAction
    timestamp createdAt
  }

  CONFIG {
    string docId PK
    boolean recommendationsEnabled
    boolean discogsEnabled
  }

Security boundary: customers can read and write only their own user profile and session subcollection. The owner can read aggregate demand signals and write operational collections. Demand signals deliberately omit user id, name, and email.
